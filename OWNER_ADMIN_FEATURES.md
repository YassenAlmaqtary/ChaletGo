# ميزات المالك والمدير - ChaletGo

## نظرة عامة

تم إعداد نظام كامل يفصل بين صلاحيات المالك والمدير بشكل احترافي:

### المالك (Owner)
- يرى **فقط** شاليهاته وحجوزات شاليهاته وتقييمات شاليهاته
- لا يرى شاليهات أو حجوزات ملاك آخرين
- لوحة منفصلة على `/owner`

### المدير (Admin)
- يرى **كل شيء** في النظام
- يستطيع حظر/إلغاء حظر أي مستخدم
- يستطيع إضافة مالك أو عميل جديد
- لوحة منفصلة على `/admin`

---

## لوحة المالك (`/owner`)

### الموارد المتاحة:

#### 1. شاليهاتي (ChaletResource)
- **الفلترة**: يرى فقط الشاليهات التي يملكها (`owner_id = Auth::id()`)
- **الإجراءات المتاحة**:
  - عرض قائمة شاليهاته
  - إضافة شاليه جديد
  - تعديل شاليهاته
  - حذف شاليهاته
  - تفعيل/إلغاء تفعيل الشاليهات
- **الحماية**: لا يمكنه الوصول لشاليهات ملاك آخرين حتى لو عرف الـ ID

#### 2. حجوزاتي (BookingResource)
- **الفلترة**: يرى فقط الحجوزات الخاصة بشاليهاته
- **الإجراءات المتاحة**:
  - عرض قائمة حجوزات شاليهاته
  - عرض تفاصيل الحجز
  - تعديل حالة الحجز (pending, confirmed, cancelled, completed)
- **الحماية**: لا يمكنه رؤية أو تعديل حجوزات شاليهات ملاك آخرين

#### 3. تقييمات شاليهاتي (ReviewResource)
- **الفلترة**: يرى فقط التقييمات الخاصة بشاليهاته
- **المعلومات المعروضة**:
  - اسم الشاليه
  - اسم العميل
  - رقم الحجز
  - التقييم (1-5 نجوم)
  - التعليق
  - حالة الاعتماد
- **الحماية**: لا يمكنه رؤية تقييمات شاليهات ملاك آخرين

### Widgets والتقارير:

#### 1. إحصائيات المالك (OwnerStatsWidget)
يعرض:
- إجمالي الشاليهات (نشط/مميز)
- إجمالي الحجوزات (مؤكد/في الانتظار)
- إجمالي الإيرادات من الحجوزات المكتملة
- متوسط التقييم
- عدد الحجوزات المكتملة

#### 2. مخطط الحجوزات (BookingsChartWidget)
- مخطط خطي يوضح عدد الحجوزات خلال آخر 6 أشهر
- يساعد المالك على تتبع أداء شاليهاته

## لوحة المدير (`/admin`)

### الموارد المتاحة:

#### 1. المستخدمين (UserResource) - محسّن
- **الإجراءات المتاحة**:
  - عرض قائمة جميع المستخدمين
  - إضافة مستخدم جديد (مدير/مالك/عميل)
  - تعديل بيانات المستخدم
  - **حظر/إلغاء حظر المستخدم** (Toggle Active)
  - حذف المستخدم
- **الفلترة**:
  - حسب نوع المستخدم (admin/owner/customer)
  - حسب الحالة (نشط/معطل)
- **المعلومات المعروضة**:
  - الاسم، البريد الإلكتروني، رقم الهاتف
  - نوع المستخدم (مع ألوان مميزة)
  - حالة الحساب (نشط/معطل)
  - عدد الشاليهات (للمالكين)

#### 2. الشاليهات (ChaletResource)
- **الفلترة**: يرى **كل** الشاليهات (لا يوجد فلترة)
- **الإجراءات**: إدارة كاملة لجميع الشاليهات

#### 3. الحجوزات (BookingResource)
- **الفلترة**: يرى **كل** الحجوزات (لا يوجد فلترة)
- **الإجراءات**: إدارة كاملة لجميع الحجوزات

#### 4. التقييمات (ReviewResource)
- **الفلترة**: يرى **كل** التقييمات (لا يوجد فلترة)
- **الإجراءات**: اعتماد/إلغاء اعتماد التقييمات

## عزل البيانات (Data Isolation)

### كيف يعمل العزل:

#### في Owner Panel:
```php
// ChaletResource
public static function getEloquentQuery(): Builder
{
    return parent::getEloquentQuery()
        ->where('owner_id', Auth::id());
}

// BookingResource
public static function getEloquentQuery(): Builder
{
    return parent::getEloquentQuery()
        ->whereHas('chalet', function (Builder $query) {
            $query->where('owner_id', Auth::id());
        });
}

// ReviewResource
public static function getEloquentQuery(): Builder
{
    return parent::getEloquentQuery()
        ->whereHas('chalet', function (Builder $query) {
            $query->where('owner_id', Auth::id());
        });
}
```

#### في Admin Panel:
```php
// لا يوجد فلترة - يرى كل شيء
public static function getEloquentQuery(): Builder
{
    return parent::getEloquentQuery();
}
```

### حماية إضافية:
- `canView()`, `canEdit()`, `canDelete()` في Owner Resources
- التحقق من `owner_id` قبل السماح بأي إجراء
- حتى لو حاول المالك الوصول لـ ID مباشرة، سيتم رفضه

## إدارة المستخدمين (Admin Only)

### إضافة مستخدم جديد:
1. اذهب إلى `/admin/users`
2. اضغط "إضافة مستخدم جديد"
3. املأ البيانات:
   - الاسم
   - البريد الإلكتروني
   - رقم الهاتف (اختياري)
   - **نوع المستخدم**: اختر (مدير/مالك شاليه/عميل)
   - كلمة المرور (8 أحرف على الأقل)
4. احفظ

### حظر/إلغاء حظر مستخدم:
#### من قائمة المستخدمين:
- اضغط على أيقونة القفل بجانب المستخدم
- تأكيد الإجراء

#### من صفحة التعديل:
- اضغط على "حظر" أو "إلغاء الحظر" في أعلى الصفحة

### حظر جماعي:
- حدد عدة مستخدمين
- اختر "حظر المحدد" أو "تفعيل المحدد"

## الأمان والحماية

### 1. عزل البيانات:
- ✅ كل مالك يرى فقط ما يخصه
- ✅ لا يمكن الوصول لبيانات ملاك آخرين
- ✅ Admin يرى كل شيء

### 2. حماية الوصول:
- ✅ `canAccessPanel()` يتحقق من نوع المستخدم
- ✅ `canView()`, `canEdit()`, `canDelete()` في كل Resource
- ✅ `getEloquentQuery()` يفلتر البيانات تلقائياً

### 3. إدارة المستخدمين:
- ✅ Admin فقط يستطيع حظر/إلغاء حظر
- ✅ Admin فقط يستطيع إضافة مستخدمين جدد
- ✅ المستخدم المعطل لا يستطيع تسجيل الدخول

## الاختبار

### اختبار عزل البيانات:
1. سجّل الدخول كمالك 1
2. أنشئ شاليه
3. سجّل الخروج
4. سجّل الدخول كمالك 2
5. **يجب ألا يرى** شاليهات المالك 1

### اختبار Admin:
1. سجّل الدخول كمدير
2. **يجب أن يرى** كل الشاليهات والحجوزات
3. جرب حظر مستخدم
4. حاول تسجيل الدخول بالمستخدم المعطل - **يجب أن يفشل**

### اختبار Widgets:
1. سجّل الدخول كمالك
2. تحقق من Widgets - **يجب أن تعرض فقط بيانات المالك**
3. سجّل الدخول كمدير
4. تحقق من Widgets - **يجب أن تعرض بيانات النظام الكاملة**

---

## ملاحظات مهمة

1. **عند حظر مستخدم**: لن يتمكن من تسجيل الدخول في أي لوحة أو التطبيق
2. **عند إضافة مالك جديد**: تأكد من تعيين `user_type = 'owner'`
3. **عند إضافة عميل**: تأكد من تعيين `user_type = 'customer'`
4. **البيانات معزولة تماماً**: كل مالك يرى فقط ما يخصه، لا يمكنه الوصول لبيانات الآخرين
